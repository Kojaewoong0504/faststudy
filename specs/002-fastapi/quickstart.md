# Quickstart: FastAPI 학습 콘텐츠 추가

## 개요
이 문서는 FastAPI 학습 콘텐츠 추가 기능의 핵심 시나리오를 검증하기 위한 빠른 시작 가이드 및 테스트 시나리오를 제공합니다.

## 전제 조건
- FastAPI 백엔드 애플리케이션이 실행 중이어야 합니다.
- MongoDB 데이터베이스가 실행 중이어야 합니다.
- 프론트엔드 애플리케이션이 실행 중이어야 합니다.

## 시나리오 1: 새로운 FastAPI 콘텐츠 추가 및 접근

### 목표
새로운 카테고리와 튜토리얼을 추가하고, 프론트엔드를 통해 해당 콘텐츠에 성공적으로 접근할 수 있는지 확인합니다.

### 테스트 단계
1. **새로운 카테고리 생성 (백엔드 API 호출)**
    - `POST /categories` 엔드포인트를 사용하여 새로운 카테고리 (예: "FastAPI 기본")를 생성합니다.
    - 응답으로 받은 `category_id`를 기록합니다.
2. **새로운 튜토리얼 생성 (백엔드 API 호출)**
    - `POST /tutorials` 엔드포인트를 사용하여 위에서 생성한 `category_id`에 속하는 튜토리얼 (예: "FastAPI 설치 및 시작하기")을 생성합니다.
    - 튜토리얼 내용은 Markdown 형식으로 작성합니다.
    - 응답으로 받은 `tutorial_id`를 기록합니다.
3. **프론트엔드에서 카테고리 목록 확인**
    - 웹 브라우저를 통해 프론트엔드 애플리케이션에 접속합니다.
    - 카테고리 목록에서 새로 추가된 "FastAPI 기본" 카테고리가 보이는지 확인합니다.
4. **프론트엔드에서 튜토리얼 목록 확인**
    - "FastAPI 기본" 카테고리를 클릭하여 해당 카테고리의 튜토리얼 목록으로 이동합니다.
    - "FastAPI 설치 및 시작하기" 튜토리얼이 목록에 보이는지 확인합니다.
5. **프론트엔드에서 튜토리얼 상세 내용 확인**
    - "FastAPI 설치 및 시작하기" 튜토리얼을 클릭하여 상세 페이지로 이동합니다.
    - 튜토리얼의 제목과 내용이 올바르게 표시되는지 확인합니다.

### 예상 결과
- 모든 API 호출이 성공적으로 완료됩니다.
- 프론트엔드에서 새로 추가된 카테고리와 튜토리얼이 정상적으로 표시되고, 상세 내용까지 확인할 수 있습니다.

## 시나리오 2: 특정 FastAPI 주제 상세 콘텐츠 조회

### 목표
사용자가 특정 튜토리얼 페이지로 직접 이동했을 때, 해당 주제의 상세 콘텐츠를 올바르게 볼 수 있는지 확인합니다.

### 테스트 단계
1. **특정 튜토리얼 ID 확인**
    - (시나리오 1에서 생성된 `tutorial_id`를 사용하거나, 이미 존재하는 튜토리얼의 ID를 사용합니다.)
2. **프론트엔드에서 직접 튜토리얼 상세 페이지 접속**
    - 프론트엔드 애플리케이션의 튜토리얼 상세 페이지 URL (예: `/tutorials/{tutorial_id}`)에 직접 접속합니다.
3. **튜토리얼 상세 내용 확인**
    - 해당 튜토리얼의 제목과 내용이 올바르게 표시되는지 확인합니다.

### 예상 결과
- URL을 통해 직접 접근한 튜토리얼 상세 페이지에서 콘텐츠가 정상적으로 로드되고 표시됩니다.
