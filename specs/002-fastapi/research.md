# Research: FastAPI 학습 콘텐츠 추가

## 데이터베이스 선택

### 결정
- **선택**: MongoDB
- **이유**:
    - **NoSQL**: 사용자가 NoSQL을 선호하며, 유연한 스키마가 콘텐츠 기반 애플리케이션에 적합합니다.
    - **쉬운 사용성**: Python 드라이버 (PyMongo, Motor)가 잘 지원되어 FastAPI와 통합이 용이합니다.
    - **확장성**: 수평적 확장이 용이하여 향후 콘텐츠 증가에 유연하게 대응할 수 있습니다.
    - **BSON**: JSON과 유사한 BSON을 사용하여 Python 딕셔너리와 쉽게 매핑됩니다.
- **대안**:
    - **DynamoDB**: AWS 생태계에 깊이 의존할 경우 좋은 선택이지만, 로컬 개발 환경 구성이 MongoDB보다 복잡할 수 있습니다.
    - **PostgreSQL (RDBMS)**: 사용자가 과하다고 판단했으며, 정형 데이터에 더 적합합니다.

## Docker 컨테이너화

### 결정
- **베이스 이미지**: `python:3.11-slim`
- **프로세스 관리자**: `uvicorn`
- **Dockerfile**: 멀티 스테이지 빌드를 사용하여 이미지 크기를 최적화합니다.
- **Docker Compose**: `docker-compose.yml`을 사용하여 FastAPI 애플리케이션과 MongoDB 데이터베이스를 함께 관리합니다.

### 모범 사례
- **.dockerignore**: 불필요한 파일 (예: `.git`, `__pycache__`)을 제외하여 빌드 속도를 높이고 이미지 크기를 줄입니다.
- **비-루트 사용자**: 보안을 위해 컨테이너 내에서 비-루트 사용자로 애플리케이션을 실행합니다.
- **환경 변수**: 데이터베이스 연결 정보 등 민감한 정보는 환경 변수로 관리합니다.

## FastAPI 학습 커리큘럼

### 목표
FastAPI의 기본부터 실무적인 내용까지 다루는 커리큘럼을 구성하여 사용자가 체계적으로 학습할 수 있도록 합니다.

### 커리큘럼 (예시)

1. **기초**
    - FastAPI 소개 및 설치
    - 첫 번째 API 만들기
    - 경로 매개변수
    - 쿼리 매개변수
2. **요청 본문**
    - Pydantic 모델을 이용한 요청 본문 정의
    - 데이터 유효성 검사
3. **응답 모델**
    - `response_model`을 이용한 응답 데이터 제어
    - 상태 코드 및 헤더
4. **의존성 주입**
    - 의존성 주입의 개념
    - 공통 로직 및 데이터베이스 연결 관리
5. **보안**
    - OAuth2와 JWT를 이용한 인증
    - API 키 기반 인증
6. **데이터베이스 연동 (MongoDB)**
    - `motor`를 이용한 비동기 데이터베이스 연동
    - CRUD 작업 구현
7. **고급 주제**
    - 미들웨어
    - CORS
    - 백그라운드 작업
    - WebSocket
8. **프로젝트 구조 및 배포**
    - 대규모 애플리케이션을 위한 프로젝트 구조화
    - Docker를 이용한 배포
